What is HTTP and HTTPS?

HTTP stands for Hyper Text Transfer Protocol. Hyper text is simply text that contains links to other text while Transfer Protocol describes how these hyper texts should be transferred. It is a fundamental part of the internet because the web is made up of endless amounts of links.

HTTP revolves around client - server communication. When a user wants to go to a website, they type in the URL which is them requesting access to the site. The server picks up this request and responds by giving the user the website they wanted. In this entire interaction, HTTP is the one carrying the user’s request and the server’s response.

The request and response cycle must follow the rules set by HTTP or else you might encounter an error such as the infamous ERROR 404.

HTTPS stands for Hyper Text Transfer Protocol Secure and is an extention of HTTP. Any data passed through HTTPS is encrypted by SSL(Secure Sockets Layout) or TSL(Transport Security Layer).

The main aim of HTTPS is to enforce a website's legitimacy and to protect the use from malicious attacks and tampering during the request and response cyle, especially when dealing with passwords, credit cards and other sensitive information. Fortunately, most webpages we have now are using HTTPS.
