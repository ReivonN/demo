What is a request?

As previously mentioned in the ‘Webpage Download’, a request is made when the user interacts with a web page by clicking links, typing in URLs or submitting forms. The final goal of the request is to gain access to an asset/resource contained within the server and this starts with the request line.

A request line is made up of three parts

  1. A one-word command known as the method which tells the server what the user wants. The two most common methods are GET and POST.
  2. The path within the URL which leads to the requested asset on the server.
  3. The version number of HTTP which shows the version that the request line tried to comply with.

As an example, a request line found in the VMware Horizon web page is GET /portal/webclient/icons-19431065/bg_image.jpg HTTP/1.1. The method for the request is GET, the path to the asset is /portal/webclient/icons-19431065/bg_image.jpg, and the version number is HTTP/1.1. By making this request, the user will then get the background image of the web page as a response.

The request line is then followed by the request header. Consisting of 0 or more headers, it is used to add additional information about the request which the server will use to give a more precise response to the user’s needs. The previous example of the bg_image has a request header of Accept-Language: en-US,en;q=0.9 . This isn’t very helpful because the request is for an image but if it was a text file and we specify a request header of Accept-Language: fr, de then we would then receive the text in French or German.

Lastly, a request can also hold a message body. The message body is optional and it will depend on the method of the request. It generally contains input data for the server to use which means that it is necessary when making a POST request. However, a GET request doesn’t need to pass any input data which makes a request body inappropriate.

Common Error Codes from requests

Error Code Description
401 Unauthorised Page unable to load due to invalid credentials and the server considers it as an unauthenticated request.
404 Not Found The page that the user is requesting is not within the server. This is commonly caused by entering an invalid URL.
500 Internal Server Error A simple error message indicating that there is something wrong with the server. Not very helpful in finding what and where the error may be.
502 Bad Gateway The server tried to communicate with another server and received an invalid response. Usually appears when the server takes too long to provide a response.
